# 设计原则

## 优秀的设计特征

1.  代码的可复用性： **与其反复从头开始，不如尽可能的重用已有的代码**
2.  可扩展性


## 设计原则-SOLID原则

1. 单一职责原则
2. 开放封闭原则
3. 里氏替换原则
4. 接口隔离原则
5. 依赖倒置原则

### 单一职责原则
*  一个类应该只有一个引起它变化的原因
*  一个类应该只负责一项任务
*  SRP 是所有原则中最简单的，但同时也是最考验功力的，对于刚接触的人来说，最大的难度就是何时该分离，要不要分离。 单一性原则并不一定要遵守，这只是一种指导思想，有时候遵守单一性原则的代码不一定是最优解

### 开放封闭原则
*  软件实体（类、模块、函数等）应该对**扩展开放，对修改封闭**
*  当需要添加新功能时，应该通过添加新代码（而不是修改已有的代码）来实现
*  这一原则的核心思想是：软件实体应该是可扩展的，而不是可修改的

### 里氏替换原则
*  子类对象应该能够替换其父类对象，而不会影响程序的正确性
*  这一原则的核心思想是：子类应该能够扩展父类的功能，而不是改变其原有功能
